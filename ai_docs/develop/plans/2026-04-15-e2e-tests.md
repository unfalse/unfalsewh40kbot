# Plan: E2E Test Suite for vox-logis-lexmechanic

**Created:** 2026-04-15
**Orchestration:** orch-2026-04-15-10-00-e2e-tests
**Status:** ⏳ Pending

## Goal

Add a complete handler-level end-to-end test suite (Vitest, no real API keys) covering all
six command/message handlers plus both utilities. "E2E" means: real handler class → real
routing logic → mocked services → verified Telegram reply. No real Telegram or external HTTP
calls in CI.

## Architecture Decisions

- **Test runner:** Vitest — fast, built-in `vi.fn()` / `vi.spyOn()`, first-class TypeScript,
  CJS-compatible via `vitest.config.ts`.
- **Mocking:** All three service interfaces (`LlmService`, `WeatherService`, `ParserService`)
  mocked with `vi.fn()`. No `vi.mock()` module hoisting needed — deps injected via constructors.
- **Grammy context:** Lightweight handmade helper in `tests/helpers/ctx.helper.ts` — builds a
  fake `Context`-shaped object cast via `as unknown as Context`. Stubs: `reply`, `replyWithHTML`,
  `replyWithChatAction`, `message`, `chat`, `from`, `me`.
- **tsconfig:** `tsconfig.test.json` extends root, sets `rootDir` to project root (`.`), adds
  `tests/**/*.ts` to `include`, to allow imports from `../src/`.
- **No `src/` changes required** — all handlers accept injected interfaces; constructors are
  already testable without modification.

## E2E-001 — Test Infrastructure Setup

**Priority:** Critical  
**Estimated time:** 1 h  
**Dependencies:** None  
**Files created/modified:**
- `package.json` — add devDependencies + `test` script
- `vitest.config.ts` — new file
- `tsconfig.test.json` — new file
- `tests/` — directory skeleton

### What to do

1. **Install dev dependencies:**
   ```
   npm install --save-dev vitest @vitest/coverage-v8
   ```

2. **Add `test` script to `package.json`:**
   ```json
   "test": "vitest run",
   "test:watch": "vitest",
   "test:coverage": "vitest run --coverage"
   ```

3. **Create `vitest.config.ts`** at project root:
   ```ts
   import { defineConfig } from "vitest/config";
   export default defineConfig({
     test: {
       globals: true,
       environment: "node",
       include: ["tests/**/*.test.ts"],
       coverage: {
         provider: "v8",
         include: ["src/**/*.ts"],
         exclude: ["src/bot.ts"],   // entry point; not unit-testable
       },
     },
   });
   ```

4. **Create `tsconfig.test.json`** at project root:
   ```json
   {
     "extends": "./tsconfig.json",
     "compilerOptions": {
       "rootDir": ".",
       "noUnusedLocals": false,
       "noUnusedParameters": false
     },
     "include": ["src/**/*.ts", "tests/**/*.ts"]
   }
   ```
   > Note: `rootDir` must be `.` (project root) so `tests/` can import from `../src/` while
   > TypeScript doesn't complain.  `noUnusedLocals/Parameters` relaxed for test files.

5. **Create directory skeleton:**
   ```
   tests/
   ├── helpers/          ← Grammy ctx helper (E2E-002)
   └── handlers/         ← one .test.ts per handler (E2E-004…009)
   └── unit/             ← url.test.ts, markdownv2.test.ts (E2E-003)
   ```

### Acceptance Criteria

- [ ] `npm test` runs without error (zero test files is OK at this stage)
- [ ] `vitest` picks up `tests/**/*.test.ts` glob
- [ ] TypeScript strict-mode compilation of `tests/` succeeds via `tsconfig.test.json`
- [ ] No `src/` files modified

## E2E-002 — Grammy Context Helper

**Priority:** Critical  
**Estimated time:** 45 m  
**Dependencies:** E2E-001  
**Files created:**
- `tests/helpers/ctx.helper.ts`

### What to do

Create a typed factory `makeFakeCtx(overrides?)` that returns an object satisfying the
`Context` interface shape (cast via `as unknown as Context`).

**Minimum shape required by all handlers:**

```ts
export interface FakeCtxOptions {
  text?: string;
  messageId?: number;
  username?: string;         // bot's own username (ctx.me.username)
  entities?: MessageEntity[]; // for SummaryTextHandler URL detection
}
```

**Stubs needed:**
- `ctx.message.text` — set from options
- `ctx.message.message_id` — default `1`
- `ctx.message.entities` — set from options (default `undefined`)
- `ctx.me.username` — set from options (default `"testbot"`)
- `ctx.reply` — `vi.fn()` resolving to `{ message_id: 999 }`
- `ctx.replyWithHTML` — `vi.fn()` resolving to `{ message_id: 999 }`
- `ctx.replyWithChatAction` — `vi.fn()` resolving to `true`

**Return type idiom:**
```ts
import type { Context } from "grammy";
// ...
return {
  message: { text, message_id: messageId, entities },
  me: { username, id: 1, is_bot: true, first_name: "Bot" },
  reply: vi.fn().mockResolvedValue({ message_id: 999 }),
  replyWithHTML: vi.fn().mockResolvedValue({ message_id: 999 }),
  replyWithChatAction: vi.fn().mockResolvedValue(true),
} as unknown as Context;
```

**Also export a service mock factory:**
```ts
export function makeMockLlm(): LlmService { ... }       // vi.fn() for wrapInPersona
export function makeMockWeather(): WeatherService { ... } // vi.fn() for getCurrentByCity, formatFactsForLlm
export function makeMockParser(): ParserService { ... }   // vi.fn() for parseUrl
```

### Acceptance Criteria

- [ ] `makeFakeCtx()` compiles under `tsconfig.test.json` strict mode
- [ ] `ctx.reply` / `ctx.replyWithChatAction` are `vi.fn()` instances (assertable in tests)
- [ ] Default options produce a valid ctx (no undefined crashes)
- [ ] Mock factories return objects whose methods are `vi.fn()` stubs

## E2E-003 — Utility Tests (url + markdownv2)

**Priority:** High  
**Estimated time:** 45 m  
**Dependencies:** E2E-001  
**Files created:**
- `tests/unit/url.test.ts`
- `tests/unit/markdownv2.test.ts`

### url.test.ts — UrlUtil

**`extractFirstHttpUrl`**
- happy: extracts first http URL from plain text
- happy: extracts first https URL
- edge: returns `undefined` for text with no URL
- edge: returns first URL when multiple present

**`assertPublicHttpUrl`**
- happy: `https://example.com` → returns URL object
- happy: `http://example.com/path?q=1` → returns URL object
- error: `localhost` → throws `"forbidden_host"`
- error: `http://127.0.0.1` → throws `"forbidden_host"`
- error: `http://0.0.0.0` → throws `"forbidden_host"`
- error: `http://::1` → throws `"forbidden_host"`
- error: `http://foo.localhost` → throws `"forbidden_host"`
- error: `http://metadata.google.internal` → throws `"forbidden_host"`
- error: `http://169.254.169.254` → throws `"forbidden_host"`
- error: `http://10.0.0.1` → throws `"private_ip"`
- error: `http://192.168.1.1` → throws `"private_ip"`
- error: `http://172.16.0.1` → throws `"private_ip"`
- error: `ftp://example.com` → throws `"bad_protocol"`
- error: `not-a-url` → throws `"invalid_url"`

**`collectUrlsFromMessage`**
- happy: entity type `"url"` → extracted from text slice
- happy: entity type `"text_link"` → `e.url` used
- happy: no entities → falls back to regex
- edge: URL in both entity list and regex → deduped (appears once)
- edge: empty text, no entities → returns `[]`

### markdownv2.test.ts — MarkdownV2

**`escape`**
- happy: plain string with no special chars → unchanged
- special chars: each char in `_*[]()~\`>#+=|{}.!-` is escaped with `\`
- backslash: `\` → `\\` (backslash escaped first)
- combined: string mixing plain text and specials → all specials escaped

### Acceptance Criteria

- [ ] All `assertPublicHttpUrl` SSRF/private-IP cases throw with correct error message
- [ ] `collectUrlsFromMessage` deduplication works
- [ ] All MarkdownV2 special chars escaped correctly
- [ ] No external network calls

**Early return — no text**

- `ctx.reply` called

  `"A &amp; B &lt;script&gt;"` in the `<b>` tag

### Acceptance Criteria

- [ ] All early-return paths produce no parser call, no reply
